# Hex Map Explorer

Portable module for the Mālama Labs Genesis 200 Hex Node Explorer. Renders the
entire globe in H3 hexes with resolution-stepping by Mapbox zoom, surfaces the
Phase 1 hex status (available / upcoming / sold / founding team), and opens a
selection panel that exposes the v3.6 marketing copy, legal disclosures, and
reservation CTA.

Designed to drop into the `launch.malamalabs.com` Next.js app (App Router).

## Setup

Dependencies are already in `apps/web/package.json` (`h3-js`, `mapbox-gl`).
No additional `npm install` is required for the components themselves.

### Generate the land-cell backdrop

The globe-wide grey decorative layer is pre-computed once. From the
`apps/web` directory:

```bash
curl -L -o src/explorer/data/ne_50m_land.geojson \
  https://raw.githubusercontent.com/martynafford/natural-earth-geojson/master/50m/physical/ne_50m_land.json

node src/explorer/scripts/generate-land-hexes.mjs
```

Outputs `data/land-cells-r1.json`, `data/land-cells-r3.json`,
`data/land-cells-r5.json`. The Resolution 5 file is the largest (~600K cells,
roughly 30 MB JSON or ~3 MB gzipped). Ship as a static asset and load on
demand at zoom 6+; consider converting to PMTiles for production.

### Generate the founding-hex manifest slice

From the `apps/web` directory:

```bash
node src/explorer/scripts/compute-founding-hexes.mjs
```

Outputs `data/founding-hexes.json` with H3 Resolution 5 indexes for the
five founding-team hexes. Already committed; rerun only if the input
addresses change.

## Color spec (dark-theme map)

| State | Fill | Opacity | Border | Interactive |
|---|---|---|---|---|
| Available | `#3b82f6` blue | 0.65 | 1px brighter blue | ✓ click → panel |
| Upcoming wave | `#3b82f6` blue | 0.25 | 1px dashed | ✗ |
| Reserved (sold) | `#dc2626` red | 0.70 | none | ✓ click → panel (read-only) |
| Reserved · Founding | `#dc2626` red | 0.70 | **2px amber `#e8b04a`** | ✓ click → panel |
| Activated | `#c4f061` brand green | 1.00 | 1px accent | ✓ click → panel |
| Future phase (rest of world) | `#3a3a3a` grey | 0.20 | none | ✗ decorative only |
| Restricted jurisdiction | `#3a3a3a` grey | 0.30 | none | ✗ |

Future-phase grey hexes have no click handler attached, no cursor change on
hover, no popover. They silently no-op. By design.

## Resolution stepping by zoom

| Mapbox zoom | H3 res rendered | What you see |
|---|---|---|
| 0 – 2.99 | Res 1 (~122 cells) | Continent-scale aggregate hexes |
| 3 – 5.99 | Res 3 (~7K cells) | Sub-regional aggregation |
| 6+ | Res 5 (~600K land cells) | Actual license cells — Phase 1 statuses light up here |

The HexMap component switches the rendered land layer as the user zooms.
Phase 1 hexes are always at Resolution 5 and only become visible at zoom 6+.

## Usage in a Next.js page

```tsx
'use client';

import { useState } from 'react';
import { HexMap } from '@/explorer/components/HexMap';
import { HexPanel } from '@/explorer/components/HexPanel';
import type { Phase1Hex, Phase1Manifest, LandCellSet } from '@/explorer/components/hex-map.types';
import manifest from '@/explorer/data/phase1-manifest.json';
import landR5 from '@/explorer/data/land-cells-r5.json';

export default function MapPage() {
  const [selected, setSelected] = useState<Phase1Hex | null>(null);

  return (
    <div style={{ display: 'flex', width: '100%', height: '100vh' }}>
      <div style={{ flex: 1, position: 'relative' }}>
        <HexMap
          accessToken={process.env.NEXT_PUBLIC_MAPBOX_TOKEN!}
          manifest={manifest as Phase1Manifest}
          landCells={{ r5: landR5 as LandCellSet }}
          onHexClick={({ hex }) => setSelected(hex)}
        />
      </div>
      {selected && (
        <HexPanel
          hex={selected}
          links={{
            erc721MetadataUrl: `/api/erc721/${selected.h3Index}`,
            cardanoReferenceNftUrl: `/api/cardano/${selected.h3Index}`,
            purchaseAgreementUrl: '/legal/hex-node-purchase-agreement',
            termsAndConditionsUrl: '/legal',
            tokenRewardsRiskUrl: '/legal/token-rewards-risk',
            zoneClassificationDocUrl: '/docs/zone-classification',
            dataDemandScoreDocUrl: '/docs/data-demand-score-methodology',
            pricingMethodologyDocUrl: '/docs/pricing',
          }}
          onReserveClick={(hex) => {
            window.location.href = `/presale?hex=${hex.h3Index}`;
          }}
          onClose={() => setSelected(null)}
        />
      )}
    </div>
  );
}
```

## Phase 1 manifest schema

`data/phase1-manifest.json` is the single source of truth for hex status
on the map. Production should serve this through a versioned API endpoint
(`/api/phase1-manifest`) that joins:

1. Static seed data (200 hex assignments — H3 index, region, classification)
2. On-chain reservation state (Cardano CIP-0068 mints + Base ERC-721 mints)
3. Operator profile (for activated hexes)

Anti-double-sale enforcement lives on-chain: one CIP-0068 asset per H3 index,
one ERC-721 token ID per H3 index, both gated by mint contracts that reject
duplicates. The manifest is downstream-reactive only — it reflects on-chain
truth, never writes it.

## Coexistence with existing `apps/web/src/components/*` files

This module is intentionally namespaced under `apps/web/src/explorer/` so
it does **not** collide with the existing components in the repo:

- `apps/web/src/components/HexMap.tsx`
- `apps/web/src/components/HexMapDynamic.tsx`
- `apps/web/src/components/MapPageClient.tsx`
- `apps/web/src/components/HexBoundaryPreview.tsx`
- `apps/web/src/components/GenesisHexDetail.tsx`
- `apps/web/src/components/GenesisHexList.tsx`

Nothing in `apps/web/src/components/` is modified by this drop-in. The
existing `/app/map/page.tsx` continues to use the existing `MapPageClient`
until someone explicitly swaps it for the explorer version below. Treat
this module as a parallel implementation to evaluate, not a hostile
replacement.

## Open items before production

1. **Seed hexes 6 – 200.** The manifest currently contains only the 5
   founding hexes. Decide the 200 cells across LA / NY / London / Tokyo /
   Idaho batches and seed the rest before launch.
2. **Wire DDS scoring.** Each Phase 1 hex needs a `dataDemandScore` value
   computed from the methodology in `/docs/data-demand-score-methodology`.
3. **Wire zone classification.** Each Phase 1 hex needs `zoneClassification`
   and `geographicMultiplier` joined from a WorldPop-derived classifier.
4. **Cross-chain mint contracts.** The "Reserve this hex" CTA is wired to
   `/presale?hex=...` but the actual mint flow (Cardano CIP-0068 + Base
   ERC-721 with anti-duplicate guards) is out of scope for this module.
5. **Mapbox token env.** Add `NEXT_PUBLIC_MAPBOX_TOKEN` to the
   `apps/web/.env` if not already present; the existing map components
   may already use a different env name — reconcile before wiring.
